# Multi-region / multi-map world model

## Purpose

`SimulationWorld` может содержать несколько регионов (`Region`). Регион — это отдельная карта/область внутри общего мира.

## Current state

- Текущая карта мира определяется как регион **Ривия**.
- `RegionId`: `rivia_region`.
- `DisplayName`: `Ривия`.
- `MapAssetId`: `rivia_region_map`.
- Ривия — первый реализованный регион мира.

Важно: Гота — портовый город внутри Ривии, но не название региона.

## Region model

Каждый регион хранит:
- стабильный кодовый идентификатор `Id`;
- отображаемое имя `DisplayName` для UI;
- идентификатор карты `MapAssetId`.

## Settlement map locations and regions

Поселение принадлежит миру как сущность, но его координаты на карте задаются через `SettlementMapLocation`, привязанную к конкретному `RegionId`.

Это означает:
- координаты `X/Y` нормализованы внутри конкретной карты региона;
- одинаковые `X/Y` в разных регионах не означают одну и ту же точку мира;
- для будущих регионов могут использоваться другие карты при сохранении общей симуляции.

## Future direction

В будущих этапах торговля, караваны и путешествия смогут функционально связывать поселения из разных регионов, оставаясь частью одного `SimulationWorld`.
